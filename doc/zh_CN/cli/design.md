# `cli` 设计

## 职责

仓库 conformance backend 的原生可执行分派器。

## 数据流

它只消费 `--backend`，原样转发其余参数，调用一个 backend adapter，并把返回码转换为进程退出状态。

## 算法与不变量

backend 名称显式且每次只分派一次；本包不拥有语料语法或数值语义。

## 失败与副作用

参数解析和进程退出属于副作用；解析、算术、分片和汇总留在被导入包中。

## 实现取舍

单一可执行入口统一了维护者工作流，但只面向仓库验证场景。

## 稳定性

本包作为仓库基础设施维护；生成声明可能随 runner 演进，不承诺下游兼容性。
